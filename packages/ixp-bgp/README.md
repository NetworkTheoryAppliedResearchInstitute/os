# ixp-bgp — NTARI OS IXP BGP Route Server

Phase 14 of the NTARI OS IXP Extension.

Wraps FRRouting (`frr`) as a ROS2 lifecycle node so it appears in the DDS
computation graph and is visible on the Globe interface.

## Files

| File | Purpose |
|---|---|
| `APKBUILD` | Alpine package definition |
| `ros2_bgp_node.py` | ROS2 lifecycle node: configure → activate → deactivate → cleanup |
| `bgp_monitor.py` | Standalone health monitor; publishes BGP state to DDS |
| `frr.conf.template` | FRRouting base config (route server semantics, bogon filters) |
| `bird.conf.template` | BIRD2 alternative config (uncomment in ixp.conf to use) |
| `bgpd.conf.template` | FRR split-daemon bgpd config (used in split-daemon mode) |

## Quick start

1. Install the package and its dependencies:
   ```
   apk add ixp-bgp
   ```

2. Edit `/etc/ntari/ixp.conf`:
   - Set `enabled = true` under `[ixp]`
   - Set `router_asn` to your real ASN
   - Set `ixp_lan_ipv4` to your peering LAN prefix

3. Enable and start:
   ```
   rc-update add ntari-ixp-bgp default
   rc-service ntari-ixp-bgp start
   ```

4. Verify BGP sessions:
   ```
   vtysh -c "show bgp summary"
   ```

5. Check DDS health topic:
   ```
   ros2 topic echo /ixp/bgp/health
   ```

## Route server semantics

The default FRR config enforces IXP route server semantics:

- `no bgp client-to-client reflection` — route server does not act as transit
- Bogon prefix filters reject RFC 1918, RFC 5737, and too-specific prefixes (/25+)
- Default route (0.0.0.0/0) is rejected on import from all peers
- Per-member prefix filters are generated by `ixp-registry` via `bgpq4`

## RPKI

Set `rpki_enabled = true` in `ixp.conf` and start `gortr` (included as a
dependency). gortr fetches the RPKI dataset from the RIPE RPKI validator and
serves it via the RTR protocol on `localhost:8282`.

## DDS topics

| Topic | Type | Content |
|---|---|---|
| `/ixp/bgp/peers` | String (JSON) | List of BGP sessions with state and prefix counts |
| `/ixp/bgp/prefix_count` | String | Total prefixes in routing table |
| `/ixp/bgp/alerts` | String | Session flap events, prefix limit violations |
| `/ixp/bgp/health` | String | `healthy` / `degraded` / `failed` |
