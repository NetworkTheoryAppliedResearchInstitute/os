# NTARI OS — IXP Member Onboarding

This guide walks through adding a new peering member to the NTARI IXP.

---

## Prerequisites

- `ntari-ixp-bgp`, `ntari-ixp-fabric`, and `ntari-ixp-registry` must be running
- The new member must have a valid ASN registered with a Regional Internet Registry
- The new member must have IRR records (ARIN, RIPE, APNIC, etc.) for their prefixes

---

## Step 1 — Add member to registry

```sh
# If the ASN is registered on PeeringDB, org name is auto-fetched:
ixp-registry-provision add --asn 64512

# Or provide org name manually:
ixp-registry-provision add --asn 64512 --org "Example Cooperative"
```

This creates a `pending` record in the member database.

---

## Step 2 — Physical connection

1. Connect the member's router to a free physical port on the IXP node.
2. Note the Linux interface name (e.g., `eth2`).
3. Verify link is up:
   ```sh
   ip link show eth2
   ```

---

## Step 3 — Assign peering IP and activate

The IXP peering LAN is defined in `ixp.conf [bgp] ixp_lan_ipv4`.
The first host address is reserved for the route server. Assign the
next available address to the new member.

```sh
ixp-registry-provision activate \
    --asn 64512 \
    --ip4 192.0.2.2 \
    --port eth2
```

This command:
1. Runs `bgpq4` to generate IRR-based prefix filters for AS64512
2. Writes prefix-list files to `/etc/frr/prefix-lists/AS64512-v4.conf`
3. Issues a `vtysh clear ip bgp soft in` to load the new filters
4. Updates the member record to `active`
5. Publishes `/ixp/members/provisioned` on the DDS graph
   (the fabric node auto-adds the port to the peering bridge)
6. Prints a welcome message to share with the member

---

## Step 4 — Configure FRR neighbor

Add the new peer session to `/etc/frr/frr.conf`:

```
neighbor 192.0.2.2 peer-group RS-CLIENTS
neighbor 192.0.2.2 description Example Cooperative (AS64512)
```

Reload without session teardown:
```sh
vtysh -c "configure terminal" -c "end" -c "write"
```

Or apply inline:
```sh
vtysh -c "conf t" \
      -c "router bgp 65000" \
      -c " neighbor 192.0.2.2 peer-group RS-CLIENTS" \
      -c " neighbor 192.0.2.2 description Example Cooperative AS64512" \
      -c "end" -c "write"
```

---

## Step 5 — Verify session

```sh
# On the NTARI IXP node:
vtysh -c "show bgp summary"

# Watch DDS topic for session state:
ros2 topic echo /ixp/bgp/peers

# Check Looking Glass:
curl http://ntari.local/lg/peer/64512
```

The new member should configure their router to peer with the route
server at the route server's peering IP (first host in the IXP LAN:
typically `192.0.2.1`).

---

## Member router configuration (reference)

FRR example for the member side:
```
router bgp 64512
 bgp router-id 192.0.2.2
 neighbor 192.0.2.1 remote-as 65000
 neighbor 192.0.2.1 description NTARI IXP Route Server
 address-family ipv4 unicast
  network 203.0.113.0/24
 exit-address-family
```

---

## Prefix filter refresh

IRR filters are generated at activation. To refresh them after a
member updates their IRR records:

```sh
ixp-registry-provision sync --asn 64512
```

This re-runs `bgpq4` and soft-resets the BGP session.

---

## Suspending a member

```sh
ixp-registry-provision suspend --asn 64512
```

This removes IRR filter files. The BGP session must also be removed
from FRR manually:
```sh
vtysh -c "conf t" \
      -c "router bgp 65000" \
      -c " no neighbor 192.0.2.2 peer-group RS-CLIENTS" \
      -c "end" -c "write"
```

---

## Member information

| Field | Where set |
|---|---|
| ASN | `ixp-registry-provision add --asn` |
| Org name | PeeringDB auto-fetch or `--org` |
| Peering IPv4 | `ixp-registry-provision activate --ip4` |
| Port | `ixp-registry-provision activate --port` |
| Prefix policy | Auto-generated by bgpq4 on activation |
| Status | `ixp-registry-provision list` |
