## Part 1: CompTIA's Troubleshooting Methodology

### CompTIA's 6-Step Framework

CompTIA defines the following best practice troubleshooting methodology:

1. **Identify the problem**
   - Gather information from users
   - Identify user changes
   - Perform backups before making changes
   - Begin documentation (update throughout)
   - Inquire about environmental/infrastructure changes

2. **Establish a theory of probable cause**
   - Question the obvious
   - Conduct external/internal research based on symptoms

3. **Test the theory to determine cause**
   - Once confirmed, determine next steps
   - If not confirmed, reestablish new theory or escalate

4. **Establish a plan of action and implement**
   - Refer to vendor instructions for guidance

5. **Verify full-system functionality**
   - Implement preventive measures if applicable

6. **Document findings**
   - Record lessons learned, actions, and outcomes

## Part 7: Risk Management Comparison

### CompTIA Risk Framework

**Problems have**:
- **Causes** (root issue)
- **Symptoms** (visible effects)
- **Consequences** (business impact)

**Priority**: Fix consequences first, then root causes.

### NTARI Implementation

**Example: "Installation too difficult" problem**

| Aspect | NTARI Approach | CompTIA Alignment |
|--------|---------------|-------------------|
| **Cause** | OS installation inherently complex | ✅ Identified in v1.3 philosophy |
| **Symptom** | Users abandon installation | ✅ Tracked in success metrics |
| **Consequence** | Empty network (no participants) | ✅ Priority #1 in roadmap |
| **Solution** | 4-tier deployment + Pi kits + installation parties | ✅ Addresses consequence AND cause |

**Assessment**: **Perfect alignment** - we solve user impact first, root causes second.
